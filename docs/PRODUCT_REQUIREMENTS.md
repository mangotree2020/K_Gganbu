## 1. 제품 개요

- **비전**: 방한 외국인이 K-Gganbu 하나로 현지 친구와 여행하듯 이동·소통·소비하게 한다.
- **북극성 지표**: 여행 1건당 쿠폰·티켓 사용 건수 (BM§7)
- **보조 지표**: 첫 실행→핵심 기능 사용 시간(10초), 통역 성공률, D7/D30 리텐션, crash-free 99.5%+
- **타깃**: ① 일본·대만 FIT(1차 테스트: 일본) ② 부산 크루즈 기항객 ③ 영어권·장기체류자

## 2. 요구사항 표기 규칙

- **ID**: `REQ-{에픽}-{번호}` — BM§6 매칭표의 PRD 열과 대응
- **우선순위(MoSCoW)**: M(Must) / S(Should) / C(Could) / W(Won't-now)
- **상태**: ✅ 구현·검증 / 🔶 코드 완료·외부설정 대기 / 🟡 부분 구현 / ⬜ 미착수
- 인수 기준(AC)은 검증 가능한 문장으로만 작성

BM 분석 결과를 반영한 재수립 원칙: ① R1 수익 기반(파트너·계측·FCM)은 그대로 최우선 ② 포인트 원장(REQ-PT)이 모든 게이미피케이션의 선행 과업이므로 R2로 앞당김 ③ 결제 라우팅은 기프티콘 혼합 결제 때문에 M 승격 ④ 챌린지·게임은 원장 완성 후(R4)로 재배치.

| 릴리스                               | 범위                                                                                                                                                 | 목표                                            |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| **R1 — 수익 기반** (즉시, ~3주)      | REQ-CP-3·4(파트너 시딩+퍼널 계측), REQ-ADM-1·2, REQ-AI-3, REQ-NT-1, REQ-PP-3 보안 요건. 병행: 기프티쇼 비즈·PG 계약 착수(O-6), 포인트 정책 확정(O-7) | E1: 쿠폰 저장→사용 30%                          |
| **R2 — 포인트 경제 v1** (~4주)       | REQ-PT-1·2·4(원장·가드레일·포인트 홈) → REQ-PD-1~3(만보기) → REQ-GS-1~4(기프티콘 스토어) + REQ-PAY-1(혼합 결제)                                      | E6: 여행 중 DAU +30% / E7: 발행 원가 600원 이하 |
| **R3 — 스탬프·등급 + 크루즈** (~4주) | REQ-ST-1~3(스탬프 투어), REQ-PT-3(등급), REQ-CR-2·3·4(크루즈 패키지), REQ-REV-1~3(리뷰 번역)                                                         | E8: 송객비 계약 1건 / E2: 하선객 설치 3%        |
| **R4 — 리텐션 확장**                 | REQ-KL-1~5(챌린지, 보상=포인트) → 측정 후 REQ-GM-1~4(전통놀이 게임)                                                                                  | E3: 귀국 코호트 D30 2배                         |
| 상시                                 | REQ-TR-3, REQ-AI-4 (변동비 KPI)                                                                                                                      | E4: API 비용 -60%                               |

> 의존성: REQ-PT-1(원장) → PD·ST·GS·KL·GM의 모든 적립 / REQ-GS-2(혼합 결제) → REQ-PAY-1 PG 계약 / REQ-ST → 파트너 확보(REQ-CP-3)와 병행.
> 크루즈(R3)는 2026 시즌(역대 최대 입항) 안에 나가야 하므로, R2가 지연되면 크루즈 패키지를 R2로 스왑한다.
