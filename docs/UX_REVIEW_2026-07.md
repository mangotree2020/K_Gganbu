> MD(머천다이징) 관점과 PO(제품) 관점의 실행 제안을 남긴다.

## 1. 검토 프레임

세 가지 질문으로 각 화면을 본다.

1. **첫 3초** — 이 화면이 무엇을 해주는지 즉시 읽히는가 (여행자는 길 위에서, 한 손으로, 급하게 쓴다)
2. **다음 행동** — 화면이 끝나는 지점에 자연스러운 다음 행동이 연결되어 있는가 (막다른 화면 금지)
3. **돈의 동선** — 이 화면에서 생긴 의사(가고 싶다/사고 싶다)가 수익 접점(쿠폰·티켓)까지 끊기지 않는가

## 2. 화면별 검토

### 2.1 온보딩 (언어 → 지역 → 관심사)

잘된 점: 가입 없이 3단계로 홈 진입하는 구조는 §14 "즉시성" 원칙에 부합한다.

개선 여지: 관심사 선택이 이후 추천(오늘의 추천 pickScore)에 반영되는 정도가 약하다. 선택한 관심사가 첫 홈 화면에서 "당신이 고른 K-Food 기준으로 골랐어요" 같은 문구로 되비쳐지지 않으면 사용자는 온보딩이 장식이었다고 느낀다.

- **PO 제안**: 온보딩 관심사 → pickScore 가중치 연결 + 첫 홈 진입 시 추천 라벨에 관심사 명시(1회). 지표: 온보딩 완료율 대비 D1 재실행.
- **MD 제안**: 관심사에 'Shopping' 선택 시 첫 홈의 딜 슬라이드를 쇼핑 카테고리 우선 정렬 — 관심사를 딜 정렬 신호로 재사용.

### 2.2 지도 (K-Map)

잘된 점: 클러스터링·카테고리 다중 필터·Naver↔Google 블렌드 슬라이더·바텀시트 리뷰까지 탐색 도구로서 완성도가 높다. 장소 탭→정보 시트→길찾기 흐름이 끊기지 않는다.

개선 여지: ① 바텀시트의 장소 정보에 그 장소의 쿠폰 유무가 보이지 않는다 — 지도는 "지금 근처" 의사결정의 마지막 화면이라 딜 연결의 가치가 홈보다 크다. ② 검색 결과가 없을 때의 빈 상태 안내가 약하다(철자·언어 문제인지 데이터 부재인지 구분 불가).

- **MD 제안 (우선)**: 바텀시트 장소 카드에 홈과 동일한 `matchDeal` 배지 적용 — 탭 시 QR 직행. "지도에서 발견 → 그 자리 할인"은 북극성 지표(여행 1건당 쿠폰 사용)에 가장 가까운 동선이다.
- **PO 제안**: 검색 무결과 시 "다른 언어로 검색해보기 / AI 깐부에게 물어보기" 두 가지 탈출구 버튼 제공. AI 연결은 무결과를 리텐션 접점으로 바꾼다.

### 2.3 통역 (Translate 탭 + Voice interpret + Conversation History)

잘된 점: 음성/카메라/회화 3모드 진입이 명확하고, 시작 카드 전체 터치·리플 애니메이션(금회 반영)으로 첫 사용 장벽이 낮다. 폴백(텍스트 번역) 탈출구 상시 노출은 preview 모델 리스크 대응으로 올바르다.

개선 여지: ① 상황별 회화(Situation Phrases)가 통역 탭 하단에 있는데, 실제 사용 맥락(식당에서 급히)에서는 스크롤이 부담 — 사용 빈도 상위 3개 상황을 칩으로 언어 스위처 아래 노출하는 편이 빠르다. ② Conversation History의 대화를 다시 보는 것 외에 활용처가 없다 — "이 대화 장소 다시 찾기"(대화 시점 위치 저장 시) 같은 재방문 연결이 없다.

- **PO 제안**: 통역 세션 종료 시 위치를 함께 저장(동의 시) → 이력에서 "이 대화를 나눈 곳" 지도 링크. 여행 후 회고(리텐션 엔진 S-3)와도 연결된다.
- **MD 제안**: 회화 카테고리 '식당' 사용 직후 "주변 식당 쿠폰 보기" 1줄 배너 — 통역 사용 맥락(지금 식당에 있음)이 가장 정확한 타겟팅 신호다.

### 2.4 AI 깐부

잘된 점: 음성 기본 모드 + 사투리 토글 + 위치·시간 컨텍스트 주입은 "현지 친구" 페르소나를 실제 경험으로 구현했다. Gemini Flash 주력 전환으로 변동비 구조도 개선됐다.

개선 여지: ① 추천 답변의 장소 카드가 지도·쿠폰으로 연결되는 계약(§18)이 지표로 관리되고 있으나, 답변 내 장소가 카드화되지 않고 텍스트로만 남는 경우가 있다 — 카드화율 자체를 지표로 봐야 한다. ② 대화 이력 모달과 헤더 X(저장 후 닫기)의 역할이 처음 쓰는 사람에게 자명하지 않다 — X를 눌러도 대화가 사라지지 않는다는 것을 첫 사용 시 1회 토스트로 알려주면 불안을 없앤다.

- **PO 제안**: `ai_reply`에 `has_card` 비율 대시보드화, 카드화 실패 응답을 주간 샘플링해 프롬프트 개선 루프를 만든다.
- **MD 제안**: 답변 장소 카드에 딜 배지 동일 적용(3번째 적용처). AI 추천은 신뢰가 실린 추천이라 배지 전환율이 목록형보다 높게 나올 가능성이 크다 — 적용처별 전환율 비교가 §7 실험 E1의 좋은 세부 지표.

### 2.5 CouTix (쿠폰·티켓·QR)

잘된 점: 세그먼트 색 전환(쿠폰 coral / 티켓 blue)·카테고리 필터·QR 화면의 캐시 즉시 발급 + 매장 정보·길찾기(금회 반영)까지 수익원 1의 소비자면은 정돈됐다.

개선 여지: ① 쿠폰 카드에 "왜 지금 써야 하는가"(urgency)가 없다 — 남은 수량·오늘 마감 표시는 전환율에 가장 직접적인 레버지만 파트너 실데이터가 선행 조건(S-1). ② 티켓이 아웃링크라는 사실이 탭 전에 보이지 않는다 — 외부 이동 아이콘을 카드에 미리 표시해 이탈감을 줄인다. ③ QR 사용 완료 후 흐름이 끊긴다 — 사용 직후가 리뷰 요청·스탬프 적립(향후 S-6)의 최적 시점이다.

- **PO 제안**: QR 사용 완료(파트너 검증 콜백) 시 "잘 다녀오셨나요? 별점 남기기" 1탭 리뷰 요청. 리뷰 실데이터 축적(홈 후기 섹션 상향 이동의 선행 조건)과 직결된다.
- **MD 제안**: 티켓 카드에 가격 옆 외부 이동 아이콘 + "예약은 파트너 사이트에서" 캡션. 신뢰 표시가 오히려 클릭을 늘린다.

### 2.6 My (프로필)

잘된 점: 크루즈 고객 토글·알림 설정·저장 쿠폰·즐겨찾기 등 설정과 자산이 한곳에 모여 있다.

개선 여지: My는 현재 "설정 화면"에 가깝다. 여행자의 자산(포인트 예정·저장 쿠폰 수·즐겨찾기 수·여행 기록)이 숫자로 요약되는 "내 여행 대시보드"로 상단을 재구성하면 재방문 이유가 생긴다. 포인트 원장(R2, REQ-PT)이 들어오는 시점과 함께 개편하는 것이 효율적이다.

- **PO 제안**: R2 포인트 원장 구현 시 My 상단을 자산 요약 카드(포인트·등급·쿠폰 n장)로 개편 — 별도 선행 작업 불필요, R2 스코프에 포함만 해두면 된다.

### 2.7 상세 모달들 (place / itinerary / cruise / emergency / allergy / tips / reviews)

잘된 점: 그라데이션 gheader 패턴이 전 화면에 통일 적용되어(금회까지 순차 반영) 화면 간 이동 시 일관된 공간감이 생겼다. place 상세의 리뷰 번역·AI 요약·Directions → 지도 경로 연결은 경쟁 우위 기능이다.

개선 여지: ① itinerary(추천 코스)의 코스 항목이 place 상세·지도와 연결되지 않는 항목이 있다 — 코스는 "계획"이므로 각 스팟이 지도 핀으로 한 번에 펼쳐지는 "코스 전체 지도 보기"가 있어야 계획 도구로 완성된다. ② emergency는 기능적으로 완성이나, 위급 상황에서 언어 전환 버튼(한국어로 보여주기)이 화면 최상단에 있어야 한다 — 폰을 상대방에게 건네는 시나리오가 핵심 사용 사례이기 때문.

- **PO 제안 (우선)**: itinerary에 "코스 전체 지도 보기" — 코스 스팟 배열을 지도 탭에 멀티 핀 + 순서 폴리라인으로 전달. 크루즈 코스(REQ-CR)와 같은 계약을 쓰면 구현 재사용이 된다.
- **PO 제안**: emergency 상단에 "Show in Korean" 고정 버튼(현재 위치보다 한 단계 위로).

## 3. 기능 Flow 종합 진단

**강한 Flow (유지)**

- 홈 추천/주변 → place 상세 → Directions → 지도 경로: 발견→이동의 본선이 끊김 없이 연결된다.
- 통역 시작 카드 → 라이브 세션 → 폴백 탈출구: 실패해도 막다른 길이 없다.
- 쿠폰 목록/홈 딜/추천 배지 → QR(캐시 즉시) → 매장 길찾기: 수익 동선의 왕복이 완성됐다.

**끊긴 Flow (보완 대상, 우선순위순)**

1. 지도 장소 시트 → 쿠폰: 지도에 딜 배지 부재 (§2.2 MD 제안 — **차기 1순위**)
2. QR 사용 완료 → 리뷰/재방문: 사용 후 흐름 부재 (§2.5 PO 제안)
3. AI 답변 장소 카드 → 쿠폰: 배지 미적용 (§2.4 MD 제안)
4. itinerary 코스 → 지도 멀티 핀: 계획 도구로서의 마감 (§2.7 PO 제안)
5. 통역/회화 사용 맥락 → 주변 딜: 맥락 타겟팅 미활용 (§2.3 MD 제안)

**공통 원칙 재확인**

- 모든 목록형 화면은 빈 상태에서 다음 행동(AI에게 묻기·지도 열기)을 제시한다 — 막다른 빈 화면 금지.
- 딜 배지·urgency·외부 이동 표시 등 상업 요소는 "정보"의 형태(할인율·마감·출처)로만 노출하고 시각 비중은 콘텐츠 아래에 둔다 — S-7의 "가치 먼저" 원칙을 화면 단위에서도 지킨다.

| M-1 | 빈 지도 탭 → 몰입 모드 토글 | 시설 없는 빈 지면 탭(Google: placeId 없는 클릭 / Naver: 최근접 시설 60m 초과) 시 검색바·필터·Blend 바·FAB·하단 시트·탭바를 숨겨 지도 전체 화면. 몰입 중 아무 지점이나 다시 탭하면 즉시 원상 복귀(조회 생략, 지연 0).                                              | Google Maps·Naver 지도의 표준 패턴. POI 탭(정보 시트)과 빈 탭(몰입)의 의도 분리     |
