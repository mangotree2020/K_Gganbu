## 2. 스토리 테스트 (정성 검증)

각 핵심 고객이 "발견 → 첫 사용 → 가치 체감 → 수익 발생"까지 끊김 없이 서술되는지 검증한다.
판정: ✅ 통과 / ⚠️ 조건부(전제 충족 필요) / ❌ 실패(스토리 단절)

### 스토리 A — 일본 FIT (유카, 29세, 부산 3박 4일) → **✅ 통과 (수익 연결은 ⚠️)**

인스타에서 부산 여행 정보를 찾다가 "지도·통역이 하나로 되는 앱" 소개를 보고 설치한다. 가입 없이 일본어 온보딩 후 홈에서 바로 돼지국밥집을 검색한다(Naver 데이터라 구글맵보다 정확). 식당에서 음성 통역으로 "내장 빼주세요"를 전달하고, 리뷰가 일본어로 보여 메뉴 선택이 쉽다. 지도에서 본 카페의 쿠폰을 저장하는 순간 LINE 로그인을 하고, QR로 10% 할인을 받는다.

- 단절 없음: 발견→설치→가치 체감(통역·지도·리뷰)→로그인→쿠폰 사용까지 자연스럽다. 현재 코드로 전 구간 동작(#8 LINE 로그인 검증 완료, #16 통역 E2E 검증 완료).
- ⚠️ 조건: **수익이 발생하려면 부산에 실제 제휴 쿠폰·티켓 인벤토리가 있어야 한다.** 앱 기능은 완료됐지만 파트너 공급(영업)이 0이면 스토리의 마지막 단락이 공백이다. → §5 전략 S-1.

### 스토리 B — 크루즈 기항객 (대만 단체 이탈 FIT, 첫 부산, 체류 8시간) → **⚠️ 조건부**

기항 전날 선내 안내지 QR로 앱을 설치한다(오프라인 대비 회화·긴급문장은 번들 탑재). 하선 후 "8시간 코스"를 AI 깐부에 요청해 감천마을→국제시장→해산물 코스를 받고, 각 장소가 지도·쿠폰과 연결된다. 승선 시간 알림을 받고 터미널로 복귀한다.

- ⚠️ 조건 1: **선내/터미널 배포 채널이 확보돼야 스토리가 시작된다.** 크루즈 관광객은 앱스토어 검색으로 오지 않는다 — CruiseYa·선사·부산항만공사 제휴가 전제. 2026년 부산 크루즈 80만 명 전망은 채널만 뚫리면 CAC가 거의 0인 최상급 시장.
- ⚠️ 조건 2: 로밍 미개통·선내 Wi-Fi 환경 → 기항 전 사전 다운로드 유도와 오프라인 코스 캐시가 필수(현재 회화·QR은 오프라인 지원, 코스 사전 캐시는 보강 필요).
- ⚠️ 조건 3: "승선 복귀 알림"은 크루즈 고객 신뢰의 핵심인데 푸시(FCM)가 외부 설정 대기 상태.

### 스토리 C — 재방문 사이클 (여행 후 리텐션) → **❌ 현재 실패 → 신규 기능으로 보완**

여행앱의 구조적 약점: 여행이 끝나면 삭제된다. 유카는 귀국 후 앱을 열 이유가 없다 → 다음 방한(대만·일본은 재방문율이 높은 시장) 때 재설치 비용·이탈이 발생하고, LTV가 1회 여행으로 끝난다.

- **한국어 챌린지 + 전통놀이 게임이 이 단절을 메우는 장치다.** 귀국 후에도 "다음 여행 준비 = 한국어 미니 학습"으로 주 1~2회 앱을 열게 하고, 챌린지·게임 보상을 **포인트(S-6 단일 원장)로 적립해 다음 여행의 기프티콘·쿠폰으로 소진**하게 하면 리텐션이 커머스 수익으로 직접 연결된다. 깐부 등급이 계정에 남아 재방문 유인을 강화한다.
- 즉 게이미피케이션은 "재미 요소"가 아니라 **LTV를 1회 여행 → 다회 여행으로 늘리는 BM 구성요소**로 정의한다. → §5 전략 S-3.

### 스토리 D — 공급자: 제휴 상점 사장 (부산 서면 식당) → **⚠️ 조건부**

외국인 손님이 늘었는데 소통·홍보 수단이 없다. K-Gganbu에 쿠폰을 올리고(파트너 등록 — `partner-coupon` Edge Function 구현됨), 외국인이 QR을 내밀면 직원이 검증한다. 월말에 사용 통계를 보고 수수료를 정산한다.

- ⚠️ 조건: **파트너용 화면(등록·검증·통계)이 별도 Admin 앱으로 미구축.** 백엔드만 있고 사장이 만질 도구가 없다 — 공급 사이드 스토리의 UI가 공백. 초기엔 운영팀 대행 등록으로 우회 가능하나 확장 불가. → §5 전략 S-1.

### 스토리 E — 포인트 경제: 걷고·방문하고·바꾼다 (유카의 여행 중 루프) → **⚠️ 조건부 (신규 설계)**

유카는 여행 첫날 2만 보를 걷고 앱이 "오늘 걸은 만큼 포인트 적립" 알림을 보낸다(일 상한까지). AI 깐부가 추천한 서면 카페 투어에서 제휴 매장 3곳의 QR 스탬프를 찍어 보너스 포인트를 받는다. 여행 마지막 날 쌓인 포인트에 소액을 결제해 CU 기프티콘을 사서 편의점 간식을 산다. 귀국 후에도 등급(깐부 레벨)이 남아 있어 다음 방한 때 포인트 부스트를 받는다.

- 이 루프는 세 가지를 동시에 해결한다: ① 여행 중 매일 앱을 열 이유(만보기 — 여행자는 하루 15,000보+를 걷는 최적 세그먼트) ② 파트너 매장으로의 실물 송객(스탬프 — 파트너 광고 상품의 근거) ③ 포인트→기프티콘 혼합 결제로 커머스 결제 습관 형성(기프티콘 마진 + 장기체류자·유학생 시장 개방).
- ⚠️ 조건 1: **포인트는 부채다.** 발행 상한·소멸 정책 없이는 캐시워크식 원가 폭탄이 된다 — 캐시워크는 광고 시청이 재원이지만 우리는 기프티콘 마진·스탬프 송객비·소멸(breakage)이 재원. §3.5 넘버 테스트로 검증.
- ⚠️ 조건 2: 기프티콘 공급은 **KT 기프티쇼 비즈(B2B) 계약**이 전제 — 매입 할인율이 마진을 결정.
- ⚠️ 조건 3: 걸음수는 클라이언트 신고 값 — 서버 검증이 불가하므로 일 상한으로 어뷰징 손실을 캡.

### 3.3 판정: **넘버 테스트 조건부 실패 → 구조 개선 필요**

**커머스 수수료 기대수익(384원) < AI·API 변동비(900~1,600원).** 현재 구조로는 사용자가 늘수록 적자가 커진다. 이것이 이 BM의 최대 리스크이며, 아래 세 가지가 모두 필요하다:

1. **변동비 절감 (목표: 1인당 500원 이하)** — POI·리뷰·번역 결과 24h+ 캐싱(리뷰 번역은 장소×언어 단위로 DB 캐시하면 사용자 간 재사용률이 매우 높음), AI 깐부 모델 티어링(단순 질문은 Haiku급), 게스트 사용량 상한(무제한은 로그인 유도로 전환), 음성통역 세션 시간 상한+텍스트 폴백 유도.
2. **전환·객단가 상승 (목표: 1인당 수익 800원+)** — 크루즈 기항 코스처럼 "시간이 없어 결정이 빠른" 고전환 상품 우선, AI 깐부 추천 카드→쿠폰 직결 동선(구현됨)의 전환 계측.
3. **인당 과금이 아닌 수익원 추가** — 크루즈 선사·여행사 B2B(1인당 서비스 피 또는 SaaS), 파트너 월 구독형 광고. 트래픽 비례가 아닌 수익으로 변동비 적자를 상쇄.

| 현재 구조      | 20,000 | 8%     | 1,200원   | 768만       | 0        | **-1,632만** |
| 비용 최적화 후 | 20,000 | 8%     | 500원     | 768만       | 0        | **-232만**   |
| 목표 구조      | 30,000 | 10%    | 500원     | 1,440만     | 800만    | **+740만**   |

→ **비용 최적화만으로도 적자 폭이 1/7로 줄고, B2B 한 축이 붙으면 흑자 전환.** 고정비(인건비 등) 제외한 공헌이익 기준이므로 실제 BEP는 B2B 계약 규모에 좌우된다.

### 3.5 포인트 경제 유닛 이코노믹스 (신규 — 만보기·스탬프·기프티콘)

**규제 회피 설계(전제)**: 포인트는 **무상 지급 마일리지로만 발행**(유상 충전 판매 없음 → 선불전자지급수단 비해당로 설계, 법무 확인 O-7), 환불·현금화 불가, 유효기간 180일, 1P = 1원.

**발행(원가) — 일 상한으로 캡**

| 적립원         | 규칙                        | 일 상한 | 여행 5일 최대 |
| -------------- | --------------------------- | ------- | ------------- |
| 만보기         | 1,000보 = 10P               | 100P    | 500P          |
| 스탬프 투어    | 제휴 매장 방문 QR 1개 = 50P | 150P    | 750P          |
| 챌린지·게임    | 데일리 미션 완료            | 30P     | 150P          |
| **합계(상한)** |                             | 280P    | **1,400P**    |

실제 평균 발행은 상한의 30~40% 수준으로 가정 → **인당 발행 원가 400~600원/여행**. 여행자 특성상 소액 잔여 포인트 소멸률이 높아(가정 40%) 실 원가는 **약 250~350원/여행**.

**회수(재원)**

| 재원           | 구조                                                                                                                                          | 인당 기대치                 |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------- |
| 기프티콘 마진  | 기프티쇼 비즈 매입 할인 — 공개 기준: API 일반상품 등급별 6~12%, 대량구매 계약 최대 27%, 상품권·금액권류 1~2%. **계획은 평균 8~10% 보수 반영** | 전환 15%×8,000원×9% ≈ 108원 |
| 혼합 결제 유도 | 기프티콘 구매 시 포인트 사용 상한 30%(등급별 상향) → 현금 결제 70%+                                                                           | 결제 습관·GMV 기여          |
| 스탬프 송객비  | 파트너가 방문당 과금하는 광고 상품(100~300원/방문, 협상 필요)                                                                                 | 방문 4회 × 150원 = 600원    |
| 소멸(breakage) | 미사용 포인트 소멸                                                                                                                            | 원가 -40% 반영 완료         |

**판정: 조건부 통과.** 스탬프 송객비가 상품화되면 포인트 경제 단독으로 손익 균형(+α)이 가능하고, 안 되더라도 인당 순비용 약 200~300원은 페이드 마케팅 CPI(3,000~5,000원)나 리인게이지먼트 광고 대비 훨씬 저렴한 **리텐션·재방문 마케팅 비용**으로 정당화된다. 단 세 가지가 깨지면 실패로 전환된다: ① 일 상한 미적용(어뷰징 원가 폭주) ② 카탈로그의 저마진 쏠림 — 상품권·금액권류는 할인율 1~2%에 그치므로 **일반상품(6~12%) 중심으로 카탈로그를 구성** ③ 포인트 유상 판매 유혹(규제 진입). 이 세 가지를 하드 가드레일로 문서화한다.

매입 할인율은 공개 기준(일반 B2B 최대 12%, 대량구매 계약 최대 27%)이 확인돼 **미지수가 아닌 협상 변수**로 격하됐다 — 월 발송액이 커지면 12~20%+ 구간 협상 여지가 있어 §3.5의 마진 가정(8~10%)에는 상방이 존재한다.

**기프티콘의 전략적 부수 효과**: 파트너 영업(REQ-CP-3)이 지연돼도 기프티쇼 카탈로그(편의점·카페·치킨 등 전국 브랜드)가 **즉시 보상 커머스 인벤토리**가 된다 — 스토리 A·C의 "인벤토리 공백" 리스크를 헤지하는 지렛대. 장기체류자·유학생(3순위 고객)에게는 주 사용 커머스가 될 수 있다.

## 5. 전략 (부족한 부분 → 실행 과제)

### S-1. 공급 사이드 구축 — 최우선

제품이 아니라 **영업이 병목**이다. ① 부산 파트너 30곳(서면·해운대·남포동 식당/카페/체험) 쿠폰 확보 — 초기 수수료 0%로 인벤토리부터 채운다. ② 파트너 Admin은 풀 앱 대신 **웹 경량판**(쿠폰 등록 + QR 검증 + 통계 1페이지, 기존 `partner-coupon` 백엔드 재사용)으로 2~3주 내 구축. ③ 티켓은 기획대로 아웃링크 우선(재고 리스크 0).

### S-2. 크루즈 B2B 채널 — 차별화 무기

2026년 부산 크루즈 80만 명은 CAC ≈ 0 채널이다. CruiseYa 연계로 ① 선내 안내지·터미널 QR 배포, ② 선사/랜드사에 "기항지 컨시어지" 1인당 피 모델 제안, ③ 기항 코스 오프라인 캐시 + 승선 복귀 알림(FCM 설정 완료 필요)을 크루즈 전용 패키지로 상품화. **커머스 전환율 검증 전에 B2B 매출을 먼저 만든다.**

### S-3. 리텐션 엔진: 한국어 챌린지 + 전통놀이 게이미피케이션 (보상은 S-6 포인트로 일원화)

여행 전(준비)~중(사용)~후(다음 여행)를 잇는 루프로 설계한다. 챌린지·게임 보상은 개별 쿠폰이 아니라 **단일 포인트 원장(S-6)에 적립**해 만보기·스탬프와 같은 경제로 묶는다 — 보상 체계가 여러 개면 사용자도 회계도 복잡해진다.

- 한국어 챌린지: 여행 실전 문장 중심(식당·택시·쇼핑 — 기존 상황별 회화 데이터 재사용), 일일 미션·연속 출석, 통역 탭에 자연 통합.
- 전통놀이 게임: 딱지치기·달고나·구슬치기 등 캐주얼 미니게임. **⚠️ 법적 경계 필수 — "오징어 게임" 명칭·비주얼·BGM은 Netflix IP이므로 사용 금지.** 전통놀이 자체(퍼블릭 도메인)를 "K-Traditional Games"로 브랜딩하고, 마케팅 문구도 "as seen in K-drama" 수준의 간접 표현까지만. 게임 승리→포인트·뱃지, 지역 랭킹(부산에서 플레이한 사람끼리) 등 여행 맥락 결합.
- 투자 순서: 챌린지 먼저(개발비 낮고 데이터 재사용), 게임은 1종 프로토타입으로 D7 리텐션 개선을 측정한 뒤 확장.

### S-4. 변동비 통제 (§3.3-1)

리뷰 번역 DB 캐시(장소×언어), AI 모델 티어링, 게스트 사용량 상한, POI 24h 캐시 강화. **1인당 여행 변동비 500원 이하**를 엔지니어링 KPI로 관리.

### S-5. 여권 자동 인식의 BM 포지셔닝

이미 구현된 여권 인식(`passport-ocr`)·택스프리(`receipt-ocr`) 흐름은 "가입 편의"가 아니라 **택스리펀드 도우미**로 포지셔닝 — 외국인의 실질 페인이자 향후 리펀드 사업자 제휴(수수료 셰어) 수익원. 단, 여권 데이터는 최고 민감정보 — 기기 내 처리 우선·서버 저장 최소화·보관기한 명시를 요구사항으로 강제(PRD REQ-PP 참조).

### S-6. 포인트 경제 구축 — 만보기·스탬프·등급·기프티콘 (신규)

모든 게이미피케이션 보상이 흐르는 **단일 포인트 원장**을 먼저 만들고, 그 위에 적립원(만보기·스탬프·챌린지·게임)과 사용처(기프티콘·쿠폰 업그레이드)를 얹는다. 원장 없이 기능별 보상을 따로 만들면 나중에 통합 비용이 커진다 — **원장이 1번 과업**이다.

- **포인트 원칙(하드 가드레일, §3.5)**: 무상 발행 전용(유상 충전 없음), 환불·현금화 불가, 유효기간 180일, 적립원별 일 상한, 서버 사이드 적립 검증(멱등키·감사 로그).
- **만보기**: `expo-sensors` Pedometer(iOS CoreMotion / Android 스텝 카운터) 기반 — 앱 실행 시 일별 걸음수 조회·적립. 캐시워크식 락스크린·상시 백그라운드 수집은 배터리·스토어 심사 리스크로 **미채택**(여행자는 어차피 매일 앱을 연다). 걸음수는 클라이언트 신고 값이므로 일 상한 100P로 어뷰징 손실을 캡.
- **스탬프 투어**: 제휴 매장 비치 QR 스캔(기존 쿠폰 QR 검증 인프라·`partner-coupon` 재사용)으로 스탬프 적립 — GPS 단독은 위조가 쉬워 QR 원칙. 테마 스탬프 카드(예: 서면 카페 5곳, 자갈치 미식 4곳) 완성 시 보너스. 파트너에게는 "방문당 과금 송객 상품"으로 판매 — S-1 파트너 영업의 두 번째 상품.
- **깐부 등급**: 누적 포인트+여행 횟수로 4단계(예: Buddy → Chingu → Gganbu → Jjin-Gganbu). 혜택은 포인트 적립 부스트(+10~30%)·기프티콘 포인트 사용 상한 상향·전용 쿠폰. 등급은 계정에 영구 귀속 — **재방문(다회 여행) LTV의 저장 장치**.
- **기프티콘 스토어(CouTix 탭 내)**: KT 기프티쇼 비즈 API로 상품 카탈로그·발권. 구매는 **포인트(상한 30%) + 카드 결제 혼합** — 결제가 필요하므로 Phase 2로 미뤘던 결제 라우팅(§24)을 앞당긴다(PRD REQ-PAY M 승격). 마진 = 매입 할인율. 파트너 쿠폰 인벤토리가 채워지기 전의 보상 커머스 공백을 메우는 헤지이기도 하다.

### S-7. 홈 머천다이징 — 섹션 구성·전환 동선 (2026-07-04 평가 확정)

홈 순서(히어로 → 퀵 타일 → 오늘의 추천 → 오늘의 딜 → 주변 추천 → 추천 코스 → 여행자 후기)는 **"가치 먼저, 수익 둘째"의 정석 구조로 유지한다.** 딜을 최상단에 두면 노출은 늘지만 "광고판 앱" 인식으로 리텐션(=장기 수익)을 깎는다 — 트리플·에어비앤비류의 에디토리얼 → 커머스 순서와 같은 원리.

| 섹션                    | 평가                                                                                         | 방침                                                                 |
| ----------------------- | -------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| 히어로                  | 개인화(위치·날씨·걸음·TTS)가 "현지 친구" 컨셉을 첫 화면에서 증명 — 강점                      | 유지                                                                 |
| 퀵 타일                 | 탭바와 중복 없는 콘텐츠 기능만 유지, CouTix HOT가 첫 타일 → 수익 진입점 확보                 | 유지                                                                 |
| 오늘의 추천 ↔ 오늘의 딜 | **인접 배치가 핵심 자산.** 추천으로 생긴 방문 의사가 그 자리에서 할인으로 이어져야 전환 발생 | ✅ 추천 카드에 해당 장소 딜 배지(탭→QR 직행) 구현 — 북극성 지표 직결 |

| 추천 코스               | 시점 의존(계획 단계 高·여행 중 低) — 여행 중 사용자 기준 중하단이 맞음                       | 유지. 온보딩 직후 첫 홈에서만 상단 노출은 실험 후보(§7)              |
| 여행자 후기             | 실데이터 축적 전까지 최하단 유지. 신뢰 요소는 전환 지점(딜) 근처에서 힘을 발휘               | 실후기 축적 후 딜 섹션 인근 이동 검토                                |

우선순위 개선(순차 실행):

1. **추천↔딜 연결 배지** — 오늘의 추천·주변 추천 카드에 해당 장소 쿠폰 배지, 탭 시 QR 발급 직행 (✅ 2026-07-04 구현 → ✅ 2026-07-05 **LBS 매칭 고도화**: 파트너 등록 시 매장 주소 지오코딩으로 partners.lat/lng 저장, 매칭은 좌표 근접(≤150m) 우선·좌표 없는 딜만 이름 폴백. 티켓도 좌표 부여로 매칭 풀 포함 — 쿠폰 없으면 티켓 배지→아웃링크)
2. 딜 urgency 표시 — 남은 수량·마감 시간 노출(파트너 실데이터 필요, S-1 이후)
3. 크루즈 고객 세그먼트에서 추천 코스를 딜 위로 스왑(기항 체류 시간 내 동선이 우선 가치) (✅ 2026-07-05 구현)
